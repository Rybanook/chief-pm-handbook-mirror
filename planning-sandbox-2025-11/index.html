Chief ‚Äî Full Build Plan (from here to ‚Äúreal Chief‚Äù)

This is the end-to-end plan to turn Chief from ‚Äúutility‚Äù into a calm, capable personal chief-of-staff. It keeps our guardrails (truth first, confirm before write, tiny diffs) and adds the brain, memory, and tools you asked for: LifeGraph, intelligent reasoning, email/SMS, reminders, rescheduling, and proactive help‚Äîwithout ever getting creepy.

North Star

  • Contract: Utterance ‚Üí Intent ‚Üí Draft ‚Üí Confirm ‚Üí Execute ‚Üí Receipt.
  • Tone: calm, concise, competent. Truth > momentum.
  • Safety: no silent writes, explicit consent, America/Vancouver TZ, reversible diffs.
  • Privacy: local-first; cloud only with flags + receipts; hard caps.

Capability Map (what ‚ÄúFull Chief‚Äù includes)

Calendar (CURRENT & expanding)

  • Create/move/cancel; de-dupe; explicit confirm; truthful replies.

Email (NEW)

  • Retrieve important mail; summarize; compose ‚Üí confirm ‚Üí send via Gmail API.
  • Smart prompts (‚ÄúWant me to reply ‚Äòrunning 5 mins late‚Äô?‚Äù) with confirm.

SMS/Texts (NEW, Android)

  • Read recent threads (permissions); compose ‚Üí confirm ‚Üí send via Intent/SmsManager.
  • Offer to text attendees on reschedule.

Reminders/Tasks (NEW)

  • Quick capture (‚Äúremind me 7am protein‚Äù), confirm prompts, reliable fire-time.

LifeGraph (NEW)

  • A local graph of People, Events, Projects, Places, Todos, Messages.
  • Ingests Calendar/Email/SMS; normalizes names/places; tracks follow-ups and commitments.

Reasoning (NEW)

  • Rules-first extractors + a constrained LLM behind a flag.
  • Planner picks tools; always yields a typed Draft; never acts without confirm.

Proactive (LATER)

  • With consent: gentle nudges (‚Äúleave now for training?‚Äù), never surprise writes.

Architecture (high level)

Android app (Compose)

  • Audio Gate (Silent/Quiet), Mic pipeline (System ASR ‚Üí Whisper), Onboarding, Brief, Chat, Confirm dialogs.
  • ASR Interface with flag: engine=system|whisper.
  • Intent Engine (rules-first), Draft UI, Tool Clients (Calendar/Email/SMS).

Backend (Node/Express)

  • /profile, /onboarding/greet, TTS; planned /calendar/*, /email/*.
  • Idempotency & de-dup; per-user store; receipts logging.

Data

  • LifeGraph (on-device primary; mirrored to backend):

  • Nodes: Person, Event, Project, Place, Message, Task.

  • Edges: attends, relates_to, located_at, mentions, assigned_to, followup_for.
  • Each item has source, confidence, seenAt, updatedAt, idempotencyKey.

Security/OAuth

  • Per-user 3-legged OAuth for Google (Gmail/Calendar) with refresh tokens‚Äîadded only after file review, one-line change summary, and scoped consent UI.

Roadmap by Phases (with hard exit criteria)

v4.0 ‚Äî Casual Talk ‚Üí Calendar

  • a.0 System ASR prototype ‚úÖ
  • a.1 Whisper NDK skeleton ‚úÖ

  • a.2 Whisper core (offline transcript, DEV flag) ‚Üí ACTIVE

  • Exit: ASR{engine=whisper, text, ms} logs; preview shows; system ASR still works when flag off.

  • b.0 Parser (rules-first) ‚Üí EventDraft

  • Relative dates (‚Äúthis Fri‚Äù), ranges (‚Äú1‚Äì2pm‚Äù), places; confidence score; missing fields list.

  • Exit: 10 utterances produce correct drafts.

  • c.0 Calendar write (safe)

  • Pre-gate: review OAuth files; one-sentence change; limited scopes; idempotency.

  • Exit: 10 varied creates/moves succeed; duplicates prevented; event IDs logged.

  • d.0 Replies (template-first)

  • Text + SSML reply that mirrors actual result; blocks speech under gate.

  • Exit: 5 scenarios (create/move/cancel/dup/invalid) produce correct replies.

v4.1 ‚Äî Polish & Reliability

  • Error taxonomy + user-facing fixes (‚Äúcouldn‚Äôt parse time‚Äîtry ‚Äòat 7pm‚Äô‚Äù).
  • Offline handling, retry, and ‚Äúsave draft‚Äù if backend absent.
  • Exit: Crash-free; receipts show clean flows.

v4.2 ‚Äî Email (retrieve + compose)

  • Read: list threads; summarize important; ‚ÄúOpen in Gmail‚Äù.
  • Compose: Draft ‚Üí Confirm ‚Üí Send (Gmail API).
  • Exit: 10 samples (short confirmations, scheduling replies) succeed; sent message ID logged.
  • Note: OAuth scope review required right before enable.

v4.3 ‚Äî SMS (retrieve + compose) (Android)

  • Read: last N messages for a contact (permission-gated).
  • Compose: Intent/SmsManager ‚Üí Confirm ‚Üí Send; delivered status where available.
  • Exit: 10 samples pass; no background sending without confirm.

v4.4 ‚Äî Reminders/Tasks

  • Quick capture; confirm; local alarm + optional Calendar task/list.
  • Exit: 10 reminders fire on time; snooze & done receipts logged.

v5.0 ‚Äî LifeGraph Foundation

  • Ingest Calendar/Email/SMS ‚Üí People/Projects/Events/Messages nodes + edges.
  • Canonicalization (merge ‚ÄúSam‚Äù = same person), provenance, confidence.
  • Surfaces: Daily Brief adds ‚Äúloose ends‚Äù (‚Äú2 unreplied emails from Sam,‚Äù ‚ÄúProject Rocking Chair: 1 open task‚Äù).
  • Exit: Graph contains expected nodes/edges after a day of normal use; queries stable.

  • Detect commitments (‚ÄúI‚Äôll send that tonight‚Äù).
  • Offer reminder/task; track until done.
  • Exit: 10 commitment cases create correct follow-ups; reminders surface in Brief.

v5.2 ‚Äî Proactive Nudges (consentful)

  • With explicit opt-in: leave-now (traffic aware later), gentle follow-ups, ‚Äúdo you want me to‚Ä¶?‚Äù prompts.
  • Exit: No writes without confirm; nudges limited and logged.

v5.3 ‚Äî Reasoning Assist (DEV flag)

  • Constrained LLM to fill missing fields into strict JSON; thresholded confidence; safe fallbacks.
  • Exit: Improves parse coverage on tricky phrasing; never bypasses confirm.

v6.0 ‚Äî ‚ÄúChief of Life‚Äù Cohesion

  • Project views (e.g., Rocking Chair): events, tasks, emails, notes in one place.
  • Weekly plan generator; recovery summaries (‚Äúwhat slipped last week?‚Äù).
  • Exit: Weekly Plan and Project rollups reliable and helpful via receipts.

Tool Contracts (so the brain can act safely)

Calendar POST /calendar/create {title,start,end,tz,location,attendees,notes,idempotencyKey}

  • Returns {eventId}; 409 on dup; idempotent on retry.

Email

  • List: /email/important?since=‚Ä¶ ‚Üí {threads:[{id,subject,from,age}]}
  • Draft: /email/draft with {to,subject,body,cc?,bcc?} ‚Üí {draftId}
  • Send: /email/send {draftId} ‚Üí {messageId}

SMS (Android)

  • Read via permissioned content resolver; send via Intent/SmsManager with explicit confirm.

Reminders

  • Local store; optional Calendar tasks; alarm manager with receipt.

Replies (NLG)

  • Template slots only; SSML via existing TTS; obey gate.

LifeGraph

  • Append-only ledger; each node/edge has source, confidence, timestamp, hash.
  • Deterministic IDs where possible; merges logged.

Reasoning & Memory

Pipeline

  • Parse rules ‚Üí fill EventDraft/EmailDraft/SmsDraft/TaskDraft.
  • If low confidence & DEV flag on ‚Üí LLM fill into strict schema.
  • Always confirm; show diffs for moves; execute; emit Receipt.
  • Memory writes both to LifeGraph and a small Facts/Decisions/Ops ledger.

Truth checks

  • TZ normalization; de-dup; idempotency; compare requested vs actual; reply mirrors actual.

Security & Privacy (non-negotiable)

  • No silent writes; confirm every side-effect.
  • OAuth scope gates: review files, state change in one sentence, then and only then merge.
  • Local-first for ASR and LifeGraph; LLM calls behind DEV flag with caps.
  • Logs redact emails/phone numbers except last 2 chars for receipts.

Risks & Mitigations

  • Auth scope creep ‚Üí Gate with file review + tiny PRs; integration tests per scope.
  • ASR latency ‚Üí Whisper tiny/en quantized; stream later; keep system ASR fallback.
  • Time parsing edge cases ‚Üí Rules first, LLM assist flagged; require confirm on low confidence.
  • Graph bloat/dup ‚Üí Canonicalization; hash-based idempotency; periodic merge jobs.

ADR-001 ‚Äî Reasoning Core v0 (‚ÄúAdvisor Sandbox‚Äù) for Chief

Status: Proposed ‚Ä¢ Owner: PM ‚Ä¢ Date: 2025-11-08 Goal: Give Chief a safe, auditable ‚Äúthinking lane‚Äù that runs prompts, stores receipts, and never mutates user data.

Decision

Add an internal Node route that shells out to our model adapter (stub by default) and writes append-only receipts to SQLite. No automations or side-effects in v0; advise-only.

API (internal only)

  • POST /_internal/advisor/run

  • body: { "domain":"health|finance|projects|‚Ä¶", "prompt":"<user/PM question>", "provider?":"stub|openai|ollama" }

  • returns (200): one JSON: {"run_id":"<ulid>", "ok":true|false, "provider":"‚Ä¶", "tokens":<n?>, "text":"<advisor output>", "receipts":{"stdout":"‚Ä¶","stderr":"‚Ä¶","ms":1234}}

Auth: same admin/ops guard you use for other internals; rate-limit; payload size cap.

Child-process call (deterministic)

  • Node spawns: python -m models.adapter_cli --provider <p> --prompt-file <tmp> (tiny CLI on top of our existing models/adapter.py; outputs one line JSON)
  • Default provider: stub (no keys).
  • OpenAI/Ollama only allowed if env creds present; otherwise return not_configured.

Data model (append-only)

Tables (SQLite):

  • advisor_runs(id INTEGER PK, run_id TEXT UNIQUE, ts TEXT, domain TEXT, provider TEXT, input_json TEXT, output_json TEXT, receipts_json TEXT, latency_ms INTEGER)
  • advisor_audits(id INTEGER PK, run_id TEXT, ts TEXT, score REAL, findings_json TEXT) No updates after insert except adding audits; never delete‚Äîrotate with the same index/rotation rules we already use.

Ops & receipts

  • Write a plain log: logs/advisor/<ts>-<run_id>.log
  • Maintain logs/advisor/index.jsonl (rotate at 5k lines / 1MB; keep last 1k) ‚Äî same policy as wrapper.
  • Add a tiny /ops/advisor/report.html generator later (reuse our HTML report skeleton).

Safety & caps

  • Max prompt length (env): ADVISOR_MAX_CHARS (e.g., 4k).
  • Max tokens (env): ADVISOR_MAX_TOKENS (e.g., 800 stub / 1,500 real).
  • v0 is read-only: advisor returns text + rationale; no writes/automations.

Rollout

  1. Add route + adapter CLI + DB tables + file logging (stub provider).
  2. Add advisor:reflect cron to write advisor_audits rows (simple rubric).
  3. Introduce domain brains by passing a domain-specific preamble into the adapter.
  4. Flip to a real provider later by env when you‚Äôre ready.

Extra reuse from what we‚Äôve already built

  • Receipts discipline: Single-line JSON everywhere; copy the wrapper‚Äôs receipts/metrics pattern for advisor runs & audits.
  • Rotation & prune: Drop-in reuse of index rotation + prune_logs for advisor logs.
  • Error budget: Reuse the counter to track ok/nonzero/timeouts for advisor calls.
  • HTML ops report: Clone ops_report.py to advisor_report.py for a quick dashboard.
  • Policy/caps mindset: Add a tiny ‚Äúadvisor caps‚Äù check (prompt len, provider allowlist) before we call the adapter.
  • Tag/branch helpers: Not needed for v0, but later we can version ‚Äúadvice packs‚Äù with annotated tags if we snapshot personas/preambles.
  • .cmd shims/Makefile: Apply the same cross-platform shims to the advisor scripts so Windows is first-class.

One sharp thought

Keep Advisor v0 advice-only until we see consistent quality; when you want ‚Äúaction,‚Äù route it through a separate ‚Äúexecutor‚Äù lane with human confirmation. That keeps trust high and blast radius low.
